DESIGN CONTEXT CAPSULE (PLAIN TEXT, AI-READY)

0) DESIGN PRINCIPLES
- Trust first, then speed
- Conversion-focused; quote module always accessible
- Clarity over cleverness; short, plain language
- Low cognitive load; generous spacing
- Accessibility baseline: WCAG AA; visible high-contrast toggle
- Mobile-first

1) BRAND VOICE AND MESSAGING
- Tone: Friendly, professional; reassuring, transparent; never pushy
- Core messages:
  "Instant quote. Friendly pickup or easy drop-off in Penrith."
  "Subject to inspection. Quote valid for 7 days."
  "Cash on pickup, or bank transfer/PayID if you prefer."
- Do: clear, calm statements (e.g., "We’ll email your quote to confirm.")
- Don’t: hype, slang, pressure

2) VISUAL IDENTITY SYSTEM
2.1 Color palette (light, trustworthy)
- Primary (Action Blue): #2563EB
- Success (Positive): #10B981
- Info/Accent (Trust): #4F46E5
- Surface (Page bg): #F8FAFC
- Card (Containers): #FFFFFF
- Text (Headings): #0F172A
- Text (Body): #334155
- Borders/Lines: #E2E8F0
- Danger (Errors): #DC2626
- Usage rules: Buttons use Primary Blue with white text; links Primary Blue with underline on hover/focus; use Success for confirmations/badges; maintain AA contrast

2.2 Typography
- Family: Inter or clean system sans
- Sizes (mobile / desktop):
  H1 hero: 28–32 / 40–48
  H2 section: 22 / 28–32
  H3 card titles: 18 / 20–22
  Body: 16 / 16–18
  Small/meta: 14
- Line-height: 1.3–1.5
- Weights: 600 headlines; 500 buttons; 400 body
- Numbers: Tabular lining for prices

2.3 Spacing and layout
- Spacing scale (px): 4, 8, 12, 16, 24, 32, 40, 48, 64
- Corner radius: ~12–16
- Shadows: soft, low elevation
- Grid: max width ~1100–1200; 12-col desktop; 1-col mobile

2.4 Iconography and imagery
- Icons: outline, rounded joins; 20–24px
- Imagery: device renders, neutral backgrounds, consistent lighting
- Alt text: literal and concise (e.g., "iPhone 13 cracked screen")

4) COMPONENT SPECIFICATIONS
4.1 Quote inputs
- Model typeahead: placeholder "Search iPhone model (e.g., iPhone 13 Pro)"; show popular families when empty
- Storage: disabled until model set; ascending values (64→1TB)
- Damage checklist: grouped sets (Screen & Face ID; Back & Cameras; Buttons & Ports; Sound & Mics; Battery; Other issues); show live count ("3 issues selected")
- Accessories: toggles (Box; Charger); helper text shown

4.2 Quote outputs
- Quote figure: large and bold; whole AUD dollars (no cents)
- Validity badge: "Valid until DD MMM, 11:59pm"
- Pickup panel: "12.4 km · ~22 min"; Fee result (Free / $30 / $XX / $50) + helper; eligibility hint if not covered

4.3 Lead form
- Inline hints on blur; friendly tone
- Photos: drag-and-drop zone; "Upload up to 3 photos"
- Consent microtext: "By continuing you agree to our Terms & Privacy. Subject to inspection."
- Submit button: disabled until required fields valid

4.4 Feedback
- Toasts: brief; bottom-center on mobile / top-right on desktop
- Banners: success/info/warn/danger with icon + one sentence
- Inline errors: under field; red + icon; do not rely on color alone

4.5 Calendar (after verification)
- Show next 7–14 days; only 12:00–20:00 slots
- "Same-day available" badge when applicable
- Slot tiles: readable; past/blocked disabled; selection shows confirmation card

4.6 Modals and drawers
- Manual review: short form; friendly copy; easy close
- Photo lightbox: full-screen; swipeable on mobile

4.7 Offline banner
- Copy: "You’re offline. We’ll save changes and retry when you’re back."
- Behavior: blocks destructive actions; auto-hide when online

5) RESPONSIVE BEHAVIOR
5.1 Breakpoints
- Mobile <480px: single column; quote first
- Tablet 480–768px: two stacked zones; quote top
- Desktop >768px: two columns; max width ~1200px
5.2 Reflow rules
- Buttons full-width on mobile; auto width on desktop
- Sticky mobile CTA appears after scrolling beyond hero: "Get my instant quote"

6) ACCESSIBILITY
- Goal: WCAG AA
- High-contrast toggle in header; persists
- Visible focus states; logical tab order
- Target sizes ≥44×44px
- Every input has visible label; placeholders never substitute labels
- Errors: text + icon; programmatically associated with fields
- Reduced motion respected; disable nonessential animations
- Alt text present and descriptive

7) MOTION AND MICRO-INTERACTIONS
- Global motion: subtle; 150–250ms durations
- Section transitions: simple fades; no parallax
- Buttons: loading spinner + "Please wait…"; minimal success pulse
- Quote change: quick count-up (~200ms) or cross-fade if reduced motion
- Checklist ticks: small fade/scale; no bounce
